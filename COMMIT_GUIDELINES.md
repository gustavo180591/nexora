# Tipos de Commit (MUY IMPORTANTE)

Usa explícitamente estos tipos de commit:

## Tipos Obligatorios

- **feat:** → nueva funcionalidad
- **fix:** → bug
- **refactor:** → código mejorado sin cambiar funcionalidad
- **style:** → UI / CSS
- **docs:** → documentación
- **chore:** → tareas internas

## Ejemplos

```bash
feat: agregar sistema de autenticación
fix: corregir error en login
refactor: optimizar consultas a base de datos
style: actualizar diseño del dashboard
docs: agregar documentación de API
chore: actualizar dependencias
```

## Formato

```
<tipo>: <descripción breve>

<descripción detallada si es necesario>
```

**IMPORTANTE:** Siempre usar uno de los tipos listados arriba.
